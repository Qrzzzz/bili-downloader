# Security Policy

## 报告安全问题

请不要在公开 issue、PR、截图或日志附件中提交任何登录态、Cookie 或账号相关信息，包括但不限于：

- `SESSDATA`
- `bili_jct`
- `DedeUserID`
- `storage_state.json`
- `cookies.txt`
- 浏览器 profile
- 可识别账号身份的日志或截图

如果你发现安全问题，请通过维护者在 GitHub 仓库中公布的私密联系方式报告，或使用 GitHub Security Advisories。公开仓库发布前，维护者应补充明确的安全联系渠道。

## 登录态和隐私

- 登录态只应保存在用户本机。
- 项目不提供云端服务。
- 项目不收集遥测数据。
- 项目不上传 Cookie、视频链接、下载记录、账号信息或日志。

## 处理原则

维护者收到报告后应尽快确认影响范围，避免在修复前公开可复现的敏感细节。修复发布后，可以在不泄露用户隐私或攻击细节的前提下说明问题类型和受影响版本。
